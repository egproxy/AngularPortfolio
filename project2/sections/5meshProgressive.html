3.4 Quadratic Simplification

In order to update the data structure for vertex splits, first before collapsing an edge, the original coordinate for v1 and half-edges originating from v0 is stored on a stack. The vertex split is done similarly to the edge collapse using the the data stored on the stack. Based on the stack information, v1 is moved to its original position, and re update connectingHalfEdge of v1 adjacency list. Due to the connectingHalfEdge face never updated after the edge collapse, it can be used to update the adjacency list for vertex split, as it was originally before the edge collapse. As it is very similar to edge collapse, the following is the inverse updates of the adjacency list:

	e->next->inv->inv=e->next;
	e->prev->inv->inv = e->prev;

Then the edge information also needs to be changed with the following:

	if(e->next->edge->halfedge[0] == e->next->inv)
		e->next->edge->halfedge[1] =e->next;
	else
		e->next->edge->halfedge[0] =e->next;

This again is similarly and the exact opposite of the edge collapse. Similarly halfedges that belongs to e->inverse->face also needs to be updated. However, if e->inverse is a nullptr, that means that the edge being split is a boundary edge. Next, using the stored information on the stack for each halfedge that is stored around v0, each halfedge points back to v0 so that it restores its original state. Then adjacent faces of v0 and v1 normal gets reupdated with the method CalcFaceNormal(i) in which i represents the faces surrounding v0 and v1. Then each vertex normal is recalculated with calc VertexNormal(i) where i is v1->next->vertex and v1->prev->vertex, as well as v0->next->vertex and v0->prev->vertex.
